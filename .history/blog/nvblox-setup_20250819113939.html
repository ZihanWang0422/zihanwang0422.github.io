NVblox Deployment Guide

April 30, 2025    Zihan Wang    Technical Tutorial

注意： 本教程基于Jetson Orin Nano平台，其他平台可能需要适当调整配置。

前置条件 (Jetson Orin Nano)

Jetson设置

确保Jetson设备已正确刷入JetPack系统，并完成基础配置。

重启Docker

sudo systemctl daemon-reload && sudo systemctl restart docker

sudo apt-get install git-lfs
git lfs install --skip-repo

环境变量设置

mkdir -p  ~/workspaces/isaac_ros-dev/src
echo "export ISAAC_ROS_WS=${HOME}/workspaces/isaac_ros-dev/" >> ~/.bashrc
source ~/.bashrc

建立环境

cd ${ISAAC_ROS_WS}/src && \
   git clone -b release-3.2 https://github.com/NVIDIA-ISAAC-ROS/isaac_ros_common.git isaac_ros_common
   
sudo apt-get install -y curl jq tar

下载NVBlox资源

NGC_ORG="nvidia"
NGC_TEAM="isaac"
PACKAGE_NAME="isaac_ros_nvblox"
NGC_RESOURCE="isaac_ros_nvblox_assets"
NGC_FILENAME="quickstart.tar.gz"
MAJOR_VERSION=3
MINOR_VERSION=2
VERSION_REQ_URL="https://catalog.ngc.nvidia.com/api/resources/versions?orgName=$NGC_ORG&teamName=$NGC_TEAM&name=$NGC_RESOURCE&isPublic=true&pageNumber=0&pageSize=100&sortOrder=CREATED_DATE_DESC"
AVAILABLE_VERSIONS=$(curl -s \
    -H "Accept: application/json" "$VERSION_REQ_URL")
LATEST_VERSION_ID=$(echo $AVAILABLE_VERSIONS | jq -r "
    .recipeVersions[]
    | .versionId as \$v
    | \$v | select(test(\"^\\\\d+\\\\.\\\\d+\\\\.\\\\d+$\"))
    | split(\".\") | {major: .[0]|tonumber, minor: .[1]|tonumber, patch: .[2]|tonumber}
    | select(.major == $MAJOR_VERSION and .minor <= $MINOR_VERSION)
    | \$v
    " | sort -V | tail -n 1
)
if [ -z "$LATEST_VERSION_ID" ]; then
    echo "No corresponding version found for Isaac ROS $MAJOR_VERSION.$MINOR_VERSION"
    echo "Found versions:"
    echo $AVAILABLE_VERSIONS | jq -r '.recipeVersions[].versionId'
else
    mkdir -p ${ISAAC_ROS_WS}/isaac_ros_assets && \
    FILE_REQ_URL="https://api.ngc.nvidia.com/v2/resources/$NGC_ORG/$NGC_TEAM/$NGC_RESOURCE/\
versions/$LATEST_VERSION_ID/files/$NGC_FILENAME" && \
    curl -LO --request GET "${FILE_REQ_URL}" && \
    tar -xf ${NGC_FILENAME} -C ${ISAAC_ROS_WS}/isaac_ros_assets && \
    rm ${NGC_FILENAME}
fi

NVBlox配置

编译和安装

cd ${ISAAC_ROS_WS}
# 拉取NVBlox包
cd src
git clone -b release-3.2 https://github.com/NVIDIA-ISAAC-ROS/isaac_ros_nvblox.git

# 编译
cd ${ISAAC_ROS_WS}
colcon build --packages-select nvblox_ros nvblox_examples_bringup

常见编译错误： 如果遇到编译错误，可能需要修改源码。详见下方的代码修改部分。

源码修改

在编译过程中可能遇到的错误需要手动修改代码。在/workspaces/isaac_ros-dev/src/isaac_ros_nvblox/nvblox_ros/nvblox_core/nvblox/include/nvblox/mapper/下面修改 multi_mapper.h，在CHECK(false) << "Requested mapping type is not implemented.";后面第89行加入return "UnknownMappingType";

RealSense Docker配置

RealSense集成

cd ${ISAAC_ROS_WS}/src && \
   git clone -b release-3.2 https://github.com/NVIDIA-ISAAC-ROS/isaac_ros_common.git isaac_ros_common

Docker容器配置

NVBlox需要在Docker容器中运行以确保环境一致性：

# 启动Isaac ROS Docker容器
cd ${ISAAC_ROS_WS}/src/isaac_ros_common && \
./scripts/run_dev.sh

容器内编译

在Docker容器内执行编译：

# 在容器内
cd /workspaces/isaac_ros-dev
colcon build --packages-select nvblox_ros nvblox_examples_bringup
source install/setup.bash

运行NVBlox

启动RealSense相机

# 在容器内启动RealSense示例
ros2 launch nvblox_examples_bringup realsense_example.launch.py

成功启动后应该能看到以下界面：

使用ROSbag数据

或者录制ROSbag后，从ROSbag运行：

ros2 launch nvblox_examples_bringup realsense_example.launch.py \
rosbag:=

功能特性

实时3D重建

  • 基于深度相机的实时体素化重建
  • 支持RGB-D传感器数据融合
  • 动态物体检测和过滤

路径规划集成

  • 与Nav2导航栈无缝集成
  • 实时代价地图更新
  • 支持多层地图表示

语义分割

  • 基于AI模型的语义理解
  • 可定制物体类别
  • 支持动态场景更新

性能优化

内存管理

NVBlox对内存要求较高，特别是在Jetson平台上：

  • 调整体素分辨率以平衡精度和内存使用
  • 设置适当的地图大小限制
  • 定期清理无效的体素数据

计算优化

  • 利用GPU加速体素化计算
  • 调整处理频率避免过度计算
  • 优化传感器数据预处理

常见问题排错

编译错误

最常见的问题是源码编译错误，需要手动修改头文件：

  • 检查CUDA版本兼容性
  • 确保所有依赖包已正确安装
  • 按照上述说明修改multi_mapper.h文件

运行时错误

  • 检查RealSense相机连接
  • 确认相机权限设置
  • 验证Docker容器网络配置

性能问题

  • 监控内存使用情况
  • 调整处理参数
  • 检查散热情况

配置参数

主要参数说明

  • voxel_size: 体素大小，影响重建精度和计算量
  • max_range: 最大检测距离
  • min_range: 最小检测距离
  • update_rate: 地图更新频率

示例配置文件

# nvblox_params.yaml
nvblox_node:
  ros__parameters:
    voxel_size: 0.05
    max_range: 10.0
    min_range: 0.5
    update_rate: 10.0
    use_color: true
    use_semantics: false

总结

NVBlox是NVIDIA提供的强大3D重建和映射工具，特别适用于移动机器人的SLAM和导航应用。通过本教程的配置，您应该能够：

  • 在Jetson平台上成功部署NVBlox
  • 集成RealSense深度相机
  • 实现实时3D环境重建
  • 与ROS2导航栈集成使用

在实际应用中，建议根据具体场景和硬件配置调整相关参数，以获得最佳的性能和重建效果。
