NVblox Deployment Guide

April 30, 2025    Zihan Wang    Technical Tutorial

Note: This tutorial is based on Jetson Orin Nano platform. Other platforms may require appropriate configuration adjustments.

Prerequisites (Jetson Orin Nano)

Jetson Setup

Ensure the Jetson device has been properly flashed with JetPack system and basic configuration is completed.

Restart Docker

sudo systemctl daemon-reload && sudo systemctl restart docker

sudo apt-get install git-lfs
git lfs install --skip-repo

Environment Variable Setup

mkdir -p  ~/workspaces/isaac_ros-dev/src
echo "export ISAAC_ROS_WS=${HOME}/workspaces/isaac_ros-dev/" >> ~/.bashrc
source ~/.bashrc

Build Environment

cd ${ISAAC_ROS_WS}/src && \
   git clone -b release-3.2 https://github.com/NVIDIA-ISAAC-ROS/isaac_ros_common.git isaac_ros_common
   
sudo apt-get install -y curl jq tar

Download NVBlox Assets

NGC_ORG="nvidia"
NGC_TEAM="isaac"
PACKAGE_NAME="isaac_ros_nvblox"
NGC_RESOURCE="isaac_ros_nvblox_assets"
NGC_FILENAME="quickstart.tar.gz"
MAJOR_VERSION=3
MINOR_VERSION=2
VERSION_REQ_URL="https://catalog.ngc.nvidia.com/api/resources/versions?orgName=$NGC_ORG&teamName=$NGC_TEAM&name=$NGC_RESOURCE&isPublic=true&pageNumber=0&pageSize=100&sortOrder=CREATED_DATE_DESC"
AVAILABLE_VERSIONS=$(curl -s \
    -H "Accept: application/json" "$VERSION_REQ_URL")
LATEST_VERSION_ID=$(echo $AVAILABLE_VERSIONS | jq -r "
    .recipeVersions[]
    | .versionId as \$v
    | \$v | select(test(\"^\\\\d+\\\\.\\\\d+\\\\.\\\\d+$\"))
    | split(\".\") | {major: .[0]|tonumber, minor: .[1]|tonumber, patch: .[2]|tonumber}
    | select(.major == $MAJOR_VERSION and .minor <= $MINOR_VERSION)
    | \$v
    " | sort -V | tail -n 1
)
if [ -z "$LATEST_VERSION_ID" ]; then
    echo "No corresponding version found for Isaac ROS $MAJOR_VERSION.$MINOR_VERSION"
    echo "Found versions:"
    echo $AVAILABLE_VERSIONS | jq -r '.recipeVersions[].versionId'
else
    mkdir -p ${ISAAC_ROS_WS}/isaac_ros_assets && \
    FILE_REQ_URL="https://api.ngc.nvidia.com/v2/resources/$NGC_ORG/$NGC_TEAM/$NGC_RESOURCE/\
versions/$LATEST_VERSION_ID/files/$NGC_FILENAME" && \
    curl -LO --request GET "${FILE_REQ_URL}" && \
    tar -xf ${NGC_FILENAME} -C ${ISAAC_ROS_WS}/isaac_ros_assets && \
    rm ${NGC_FILENAME}
fi

NVBlox Configuration

Build and Installation

cd ${ISAAC_ROS_WS}
# Pull NVBlox package
cd src
git clone -b release-3.2 https://github.com/NVIDIA-ISAAC-ROS/isaac_ros_nvblox.git

# Build
cd ${ISAAC_ROS_WS}
colcon build --packages-select nvblox_ros nvblox_examples_bringup

Common Build Errors: If you encounter build errors, you may need to modify the source code. See the code modification section below.

Source Code Modification

Errors that may be encountered during compilation require manual code modification. In /workspaces/isaac_ros-dev/src/isaac_ros_nvblox/nvblox_ros/nvblox_core/nvblox/include/nvblox/mapper/, modify multi_mapper.h by adding return "UnknownMappingType"; on line 89 after CHECK(false) << "Requested mapping type is not implemented.";

RealSense Docker Configuration

RealSense Integration

cd ${ISAAC_ROS_WS}/src && \
   git clone -b release-3.2 https://github.com/NVIDIA-ISAAC-ROS/isaac_ros_common.git isaac_ros_common

Docker Container Configuration

NVBlox needs to run in a Docker container to ensure environment consistency:

# Start Isaac ROS Docker container
cd ${ISAAC_ROS_WS}/src/isaac_ros_common && \
./scripts/run_dev.sh

Build in Container

Execute compilation inside the Docker container:

# Inside the container
cd /workspaces/isaac_ros-dev
colcon build --packages-select nvblox_ros nvblox_examples_bringup
source install/setup.bash

Running NVBlox

Start RealSense Camera

# Start RealSense example inside container
ros2 launch nvblox_examples_bringup realsense_example.launch.py

Using ROSbag Data

ros2 launch nvblox_examples_bringup realsense_example.launch.py \
rosbag:=

Features

Real-time 3D Reconstruction

  • Real-time voxelized reconstruction based on depth cameras
  • Support for RGB-D sensor data fusion
  • Dynamic object detection and filtering

Path Planning Integration

  • Seamless integration with Nav2 navigation stack
  • Real-time cost map updates
  • Support for multi-layer map representation

Semantic Segmentation

  • AI model-based semantic understanding
  • Customizable object categories
  • Support for dynamic scene updates

Performance Optimization

Memory Management

NVBlox has high memory requirements, especially on Jetson platforms:

  • Adjust voxel resolution to balance accuracy and memory usage
  • Set appropriate map size limits
  • Regularly clean up invalid voxel data

Computational Optimization

  • 利用GPU加速体素化计算
  • 调整处理频率避免过度计算
  • 优化传感器数据预处理

常见问题排错

编译错误

最常见的问题是源码编译错误，需要手动修改头文件：

  • 检查CUDA版本兼容性
  • 确保所有依赖包已正确安装
  • 按照上述说明修改multi_mapper.h文件

运行时错误

  • 检查RealSense相机连接
  • 确认相机权限设置
  • 验证Docker容器网络配置

性能问题

  • 监控内存使用情况
  • 调整处理参数
  • 检查散热情况
  • Enable GPU acceleration for voxel operations
  • Adjust processing thread count based on CPU cores
  • Use asynchronous processing to reduce latency

    • Configuration Parameters

    Main Parameter Descriptions

    • voxel_size: Voxel size, affects reconstruction accuracy and computational load
    • max_range: Maximum detection distance
    • min_range: Minimum detection distance
    • update_rate: Map update frequency

    Example Configuration File

    # nvblox_params.yaml
nvblox_node:
  ros__parameters:
    voxel_size: 0.05
    max_range: 10.0
    min_range: 0.5
    update_rate: 10.0
    use_color: true
    use_semantics: false

    Summary

    NVBlox is a powerful 3D reconstruction and mapping tool provided by NVIDIA, particularly suitable for SLAM and navigation applications of mobile robots. Through this tutorial configuration, you should be able to:

    • Successfully deploy NVBlox on Jetson platform
    • Integrate RealSense depth cameras
    • Achieve real-time 3D environment reconstruction
    • Integrate with ROS2 navigation stack

    In practical applications, it is recommended to adjust relevant parameters based on specific scenarios and hardware configurations to achieve optimal performance and reconstruction results.
